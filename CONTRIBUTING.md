# 投稿指南

欢迎把你用 Vibe Coding 做出来的作品加进来！两种方式，任选其一。

## 方式一：提 Issue（推荐）

点 [投稿表单](../../issues/new?template=submit-project.yml)，填几项就行（产品名、链接、一句话介绍、用了哪些 AI 工具、是否本人）。维护者会帮你整理进清单。适合不熟悉 Git 的同学。

## 方式二：直接发 Pull Request

1. Fork 本仓库。
2. 打开 `README.md`，找到「作品清单」。
3. 如果**当月还没有小节**，新增一节，放到清单最上面：
   ```markdown
   ### 2026 年 7 月

   #### [你的名字](https://github.com/你的用户名 或 你的主页)
   * :white_check_mark: [产品名](https://链接)：一句话介绍。🛠 Cursor · [仓库](https://github.com/...)
   ```
   如果当月已有小节，直接在你自己的 `#### 名字` 下加条目即可。
4. 提交 PR，标题写 `投稿: 产品名`。

## 入选标准（提交前自查）

- ✅ 是**能打开、能用的产品**（网站 / App / 小程序 / 桌面 / 插件），有可访问链接。
- ✅ **独立/个人开发者**作品（一个人或很小的团队）。
- ✅ 开发过程**大量借助 AI 编程**（Vibe Coding）。
- ✅ 面向**中文圈**（中文开发者，或主要服务中文用户）。
- ❌ 不收：教程、课程、资源合集（awesome 类）、纯代码库/框架、纯 API/开发者工具。

> 不确定算不算？先提 Issue 问一句就好。

## 写好那一句介绍

一句话介绍是别人**点不点进去**的关键。请写**具体**：

- ✅ 好：「本地优先的家庭健康档案库，AI 帮你读懂化验单，数据不出本机。」
- ❌ 差：「一个很好用的健康管理工具。」

可选加分项：
- 🛠 注明用了哪些 AI 编程工具（Cursor / Claude Code / Codex / Copilot / Trae…）。
- 开源的话附上仓库链接。
- 状态标记：`:white_check_mark:` 在运营 / `:clock8:` 开发中 / `:x:` 已关闭。

## 一点约定

- 一次 PR 尽量只加**你自己的**作品；帮别人代投请注明。
- 链接必须可访问；已关站的老作品可标 `:x:` 保留，但新投稿请确保能打开。
- 保持友善。这里是给独立开发者互相看见、互相打气的地方。
